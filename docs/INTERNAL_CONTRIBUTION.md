# Internal Content Contribution Guide

This guide is for CSA Calgary board members, volunteers, and content contributors who need to update the website without working deeply in code.

## Updating the Site

Use the admin for normal page and event edits:

```text
https://csacalgary.org/admin/
```

The admin uses GitHub sign-in. You need approved repository access.

The admin supports:

- Events
- Home
- About
- Get Involved
- Contact

Ask the website coordinator for board, sponsor, resource, navigation, form, or layout changes.

## Adding an Event

1. Go to `/admin/`.
2. Sign in with GitHub.
3. Open `Events`.
4. Select `New Events`.
5. Complete the event fields.
6. Save and publish.
7. Check the live page after deployment.

## Event Details to Prepare

- Title
- Date and time with timezone
- Short description
- Full event description
- Location
- Venue and address
- Capacity, if known
- Event type, such as Meetup, Workshop, Panel, Summit, Training, or Networking
- RSVP link
- LinkedIn event link
- Agenda
- Approved image, if available

RSVP and LinkedIn links may be blank while pending; the site shows a coming-soon state.

## Event Copy Template

```markdown
Calgary's cloud security community is gathering for [topic].

This session will cover [main topics]. Attendees will leave with [practical takeaways].

**Who should attend:** [Audience.]

**What to bring:** [Questions, laptop, ID, or other notes.]
```

## Approval Checklist

- Names, titles, dates, and links are correct.
- Venue information is approved for publication.
- Speakers approved their names and bios.
- Sponsors approved public recognition.
- Images are approved for web use.
- No private phone numbers or personal email addresses are included.

## Request Template


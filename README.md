# CSA Calgary Chapter Website

This is the official website for the Cloud Security Alliance Calgary Chapter.

CSA Calgary is a local professional chapter connected to the global Cloud Security Alliance community. The chapter exists to advance cloud security knowledge and trusted cloud adoption in Calgary and surrounding communities through education, community engagement, and research-aligned activities.

The website is the chapter's public home for events, community participation, board information, sponsorship, contact details, and cloud security resources.

## What the Site Communicates

The site should help visitors quickly understand:

- Who CSA Calgary is
- What the chapter exists to do
- Which events are coming up
- How to volunteer, speak, sponsor, or attend
- Who serves on the Board of Directors
- Where to find trusted CSA and cloud security resources
- How to contact the chapter

## Primary Audience

The website is written for:

- Cloud security practitioners
- Security engineers and architects
- IT, risk, governance, audit, and compliance professionals
- Technology leaders adopting or operating cloud platforms
- Students and early-career professionals interested in cloud security
- Sponsors, speakers, venue partners, and volunteers
- Members of the broader Calgary and Alberta cybersecurity community

## Chapter Positioning

CSA Calgary should be presented as:

- Local and community-first
- Practical rather than theoretical
- Vendor-neutral
- Professionally credible
- Connected to CSA global research, training, certifications, and working groups
- Open to practitioners, leaders, researchers, students, partners, and volunteers

The preferred tone is clear, professional, useful, and grounded in real cloud security practice.

## Main Site Sections

### Home

The home page introduces the chapter and its mission:

> Advancing cloud security leadership in Calgary.

It highlights the chapter's focus on learning, connection, and contribution. It also surfaces upcoming events when available.

### About

The About page explains CSA Calgary's purpose, mission, objectives, and connection to the global Cloud Security Alliance.

Core mission:

> To advance cloud security knowledge and trusted cloud adoption in Calgary and surrounding communities by delivering education, community engagement, and research-aligned activities.

### Board of Directors

The board page introduces the volunteer leaders helping establish and grow the chapter.

Board profiles should include:

- Name
- Chapter role
- Approved photo
- LinkedIn URL when available
- Short bio focused on chapter responsibility

### Events

The events section publishes upcoming and past chapter sessions. Events may include:

- Technical talks
- Practitioner panels
- Workshops
- Networking sessions
- Community meetups
- Training-oriented sessions

Event pages should answer:

- What is the event?
- Who should attend?
- When and where is it happening?
- How does someone RSVP?
- What will attendees learn or discuss?
- Is there an agenda?

## Content Standards

All public website content should be:

- Accurate
- Concise
- Professional
- Vendor-neutral
- Community-oriented
- Useful to a Calgary or Alberta cloud security audience

Avoid:

- Sales language
- Unverified claims
- Private contact details
- Overly long biographies
- Publishing speaker, sponsor, or venue details before approval
- Images without permission for public website use

## Event Publishing Standards

Before an event is published, confirm:

- Event title
- Date and time, including timezone
- Venue name and address, if public
- RSVP link, if available
- LinkedIn event link, if available
- Speaker approvals
- Sponsor approvals, if sponsors are named
- Short description
- Target audience
- Agenda, if confirmed

RSVP and LinkedIn links may be left blank while details are pending. The site will show a coming-soon state.

The CSA Calgary website should be maintained by the chapter's website coordinator with input from the board, program team, sponsorship lead, volunteer lead, and event organizers.

Content should be reviewed for accuracy before publishing, especially dates, names, titles, sponsor references, and external links.
